# ■ プロジェクト概要
テーマ：
AWS SAA（Solutions Architect Associate）向けの「AI解説付き問題集アプリ」の開発
目的：
・AWS SAAの学習理解を深める
・AI（ChatGPT / Claude）を活用した開発プロセスを記録する
・ポートフォリオとして活用する

# ■ 開発方針
・まずはMVP（最小構成）で開発する
・小さく作って改善する
・AIとのやり取りも含めて記録する

# ■ 要件定義

## ① アプリの目的・ゴール
・対象ユーザー：AWS SAAを学習する初心者
・解決したいこと：問題を解くだけでなく、AI解説で理解を深められるようにする
・MVPの完成条件：IAMカテゴリの問題を5問解けて、回答後に解説が表示される

## ② 機能要件
やること
・問題を表示できる
・選択式で回答できる
・正解・不正解を判定できる
・回答後にGemini APIを使って解説をリアルタイムで生成・表示できる

やらないこと
・ログイン機能
・学習履歴の保存
・問題の自動生成
・複数カテゴリ対応

## ③ 非機能要件
・対応端末：まずはPCブラウザ
・画面表示：大きな待ち時間なく動くこと
・セキュリティ：個人情報を扱わないので最小限でよい
・API呼び出しに数秒かかることを許容する
・APIエラー時はエラーメッセージを表示する
・APIキーはコードに直書きしない（.envファイルで管理）

## ④ データ設計
解説はGemini APIで都度生成するため、JSONにexplanationは持たない
```json
{
  "id": 1,
  "question": "問題文",
  "choices": ["選択肢A", "選択肢B", "選択肢C", "選択肢D"],
  "answerIndex": 0,
  "category": "IAM"
}
```
※ answerIndexは0始まりとする

## ⑤ 画面設計
必要な画面
・問題表示画面
・回答結果・解説表示画面

ユーザーの操作フロー
1. 問題が表示される
2. 選択肢を選ぶ
3. 回答する
4. 正解・不正解が表示される
5. 「Geminiで解説を生成中...」が表示される
6. 解説が表示される
7. 次の問題へ進む

## ⑥ 技術スタック・実装方針
・言語：Python
・フレームワーク：Streamlit
・データ管理：JSONで管理（DBは使わない）
・AI API：Gemini API（解説生成のみに使用）
・SDK：google-genai（pip install google-genai）
・APIキー管理：.envファイル
・まずはローカルで動けばOK

# ■ 基本設計

## ① システム構成
ユーザー → Streamlitアプリ → JSON読み込み → 回答 → Gemini API → 解説表示

## ② 画面構成（1画面完結）
・タイトル（AWS SAA 問題集）
・カテゴリ表示（IAM）
・問題番号（例：1 / 5 + 進捗バー）
・問題文
・選択肢（ラジオボタン）
・回答ボタン
・正解・不正解表示
・ローディング表示（解説生成中）
・AI解説表示
・次の問題へボタン

## ③ 画面動作フロー
1. アプリ起動
2. JSONから問題を読み込む
3. 問題を表示する
4. ユーザーが選択肢を選ぶ
5. 回答ボタンを押す
6. 正解・不正解を表示
7. 「Geminiで解説を生成中...」を表示
8. Gemini APIを呼び出す
9. 解説を表示
10. 次の問題へ進む

## ⑤ モジュール設計
app.py
・画面表示
・ユーザー入力処理
・正誤判定
・gemini_client.py の呼び出し

services/gemini_client.py
・Gemini APIとの通信処理
・プロンプト生成
・解説テキストの取得

## ⑥ Gemini API連携設計
・呼び出しタイミング：回答ボタン押下後
・渡す情報：問題文・選択肢・正解・ユーザーの回答
・プロンプト内容：初心者向けに正解の理由と他選択肢が違う理由を説明する
・出力：初心者向け解説文

## ⑦ UI/UX設計
ローディング表示
・「Geminiで解説を生成中...」を表示

エラー時
・API失敗：「解説の取得に失敗しました。再試行してください。」を表示
・JSON読み込み失敗：エラーメッセージを表示してアプリを停止

## ⑧ 状態管理（Streamlit）
・現在の問題番号（index）
・選択された回答
・回答済みフラグ
※ st.session_stateで管理する

## ⑨ スコープ
・IAMカテゴリのみ
・問題数：5問
・ローカル環境のみ
・ユーザー管理なし
